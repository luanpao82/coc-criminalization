FY2024 is the largest year because HUD folded the Youth Homeless Demonstration Program (YHDP) into the main competition. Eight CoCs submit DOCX instead of PDF; nine require OCR and are held out of the analytic sample.

2 · Extraction pipeline

  1. File inventory (build_file_inventory.py) — indexes every source file, detects scanned PDFs, flags duplicate PDF/DOCX pairs, normalizes CoC IDs.
  2. Format-aware adapters — native-PDF, OCR'd PDF, and DOCX each implement the same output schema. Every cell carries source page, bbox, and extractor version as provenance.
  3. Codebook & crosswalk — 331 canonical FY2024 variables mapped across FY2022/23 question IDs. Wording changes flagged explicitly (see the DV measurement-invariance story on the design page).
  4. Anchor-then-parse — every question is located by its 1B-1. / 1D-4.-style anchor, and only the local block is parsed. A parse failure in one block cannot corrupt neighboring variables.
  5. Controlled-vocabulary validation — categorical outputs are checked against the codebook. Out-of-vocab values get needs_review=True; they are never silently coerced.
  6. Iteration loop — extractor diffs against the manual FY2024 spreadsheet, errors classified into six categories, pipeline refined until convergence.

Panel-safety categorization:

  • panel_safe — present in ≥50% of CoCs in every year. Used directly in the primary regression.
  • mostly_panel — present in two of three years. Used with care.
  • year_specific — asked in only one year. Cross-sectional use only.
  • sparse — free-text narrative fields awaiting Stage-2 LLM coding.

4 · Review queue

399 value disagreements and 1,131 auto-fill candidates are staged for PI adjudication.

  • Value mismatches — both the manual spreadsheet and the extractor have a value, and they disagree. Every row cites a specific PDF page; when the extractor matches the source, the manual entry is treated as the one to correct.
  • Auto-fill candidates — the manual cell is blank; the extractor has a value read directly from the PDF. A spot-check of ~30 random rows is recommended before backfilling the spreadsheet.

The pipeline does not auto-patch the manual file. Every proposed backport goes through PI review first, preserving provenance.

5 · Outstanding data work

  1. Nine CoCs need OCR. GA-508, MD-600, MI-508, NJ-511, NV-502, PA-501, TX-624, VI-500, WA-502 ship as scanned or font-encoded PDFs. ocrmypdf infrastructure is in place; the task has not been run.
  2. 41 narrative variables not yet coded. Stage-2 LLM coding with verbatim quotation is specified but not rolled out (~$50–60 in API cost). None of these are in the primary DV.
  3. Formal 20-CoC stratified audit. A per-coder cross-check against the PDFs is outstanding; this is the last blocker on publishing a clean reference dataset alongside the automated one.
